# F3 — Speaker Extraction

## Feature Description

Automatically extract speaker names from meeting notes and count how many times each speaker appears. Uses a simple regex pattern: lines starting with a letter, followed by word characters/spaces/hyphens, ending with a colon.

---

## Test Case 1 — Multiple Speakers, Repeated

| Field | Value |
|-------|-------|
| Input | `Alice:\nIssue A\n\nBob:\nIssue B\n\nAlice:\nIssue C` |
| Expected Result | `Alice -> 2`, `Bob -> 1` |

---

## Test Case 2 — Multi-word Speaker Names

| Field | Value |
|-------|-------|
| Input | `Alice Smith:\nIssue A\n\nBob Lee:\nIssue B` |
| Expected Result | `Alice Smith -> 1`, `Bob Lee -> 1` |

---

## Test Case 3 — No Speaker Headers

| Field | Value |
|-------|-------|
| Input | `Meeting notes with no speaker headers` |
| Expected Result | No speakers extracted; application remains stable. |

End of F3 test cases.
