# F2 — Meeting History

## Feature Description

Users can view all previously saved meeting notes. Meetings are loaded from SQLite and displayed with newest first. Each entry shows the meeting ID, creation timestamp, and full text content.

---

## Test Case 1 — Single Meeting

| Field | Value |
|-------|-------|
| Precondition | At least one meeting has been saved via the F1 upload form. |
| Action | Scroll to the **Meeting History** section below the upload form. |
| Expected Result | The saved meeting appears in the history list with its ID, creation time, and text content. |

---

## Test Case 2 — Multiple Meetings (Newest First)

| Field | Value |
|-------|-------|
| Precondition | Three meetings have been saved (e.g., via manual text, TXT upload, and PDF upload). |
| Action | Scroll to the **Meeting History** section. |
| Expected Result | All three meetings appear in the list. The most recently saved meeting is at the top. |

---

## Test Case 3 — Empty Database

| Field | Value |
|-------|-------|
| Precondition | No meetings have been saved (fresh database or all records deleted). |
| Action | Scroll to the **Meeting History** section. |
| Expected Result | An informational message is displayed: "No meeting notes saved yet. Use the form above to add one." No errors occur. |

End of F2 test cases.
