# F4 — Action Item Extraction

## Feature Description

Automatically extract action items from meeting notes using keyword-based detection. Recognizes lines starting with "Action item:", "TODO:", "Task:", or "Action:" and captures the following content (same line or next non-empty line).

---

## Test Case 1 — Action Item Keyword

| Field | Value |
|-------|-------|
| Input | `Charlie:\nAction item:\nCreate index on customer_email.` |
| Expected Result | One action item: "Create index on customer_email." |

---

## Test Case 2 — Multiple Keywords

| Field | Value |
|-------|-------|
| Input | `TODO:\nPrepare proposal.\n\nTask:\nReview architecture.` |
| Expected Result | Two action items: "Prepare proposal." and "Review architecture." |

---

## Test Case 3 — No Action Keywords

| Field | Value |
|-------|-------|
| Input | `Regular discussion text with no action keywords.` |
| Expected Result | No action items extracted; application remains stable. |
---
End of F4 test cases.
