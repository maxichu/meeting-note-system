# F5 — Owner Detection

## Feature Description

Automatically detect action item owners using a two-pass rule cascade: (1) explicit "Owner:" or "Assigned to:" declarations, (2) nearest preceding speaker header. Falls back to "Unassigned" if neither is found.

---

## Test Case 1 — Explicit Owner Declaration

| Field | Value |
|-------|-------|
| Input | `Owner: Alice\nAction item:\nCreate database index.` |
| Expected Result | Owner = "Alice" |

---

## Test Case 2 — Nearest Preceding Speaker

| Field | Value |
|-------|-------|
| Input | `Bob:\nAction item:\nReview architecture.` |
| Expected Result | Owner = "Bob" |

---

## Test Case 3 — No Owner Found

| Field | Value |
|-------|-------|
| Input | `Action item:\nPrepare proposal.` |
| Expected Result | Owner = "Unassigned" |

End of F5 test cases.
